25/03: WOT IS GENERATIVE DESIGN?


In class last week we thought about Instructional and Algorithmic art, including examples from the likes of Sol Ewitt and Yoko Ono. We thought about how we could produce a p5 'sketch' following their instructional poems.

Created by Lauren McCarthy, JavaScript is a general-purpose programming language with a broad range of applications, while p5 is a JavaScript library specialized for creative coding, graphics, and interactive projects, offering a simplified and user-friendly approach to web development.

In-class we made a dice-rolling-shape-drawing game. The parameters were: two dice are rolled as many times you like, resulting in up to 10 shapes being drawn. See Dice Game Instructions. This excercise was fun and quick, and we were then instructed to translate the rules from class into p5, with the following tips: Include a variable (declare at top with var diceRoll; and use later such as diceRoll=random(1, 6); to simulate a dice roll, &/or create an array(list) for your colours and shapes, and see how accurately you can translate the rules.

I really struggled with wrapping my head around how this would work. Would the user interact with the game, pressing the button to rolling the dice, and generate numbers to draw the shapes? Or, would they watch while this pre-programmed code rolled random numbers to produce shapes? This would be the difference of user-interaction.. I decided that the user would press a button, generating a number/6 and a number/12 (dice roll). After three rolls, the sketch would draw a shape; the colour, size, and position determined by the rolls. I still did not know how I would go about coding this, but I happy to have conceptualised it, and excited to learn more...
